ТРЕБОВАНИЯ К ПРОГРАММАМ "ПРАКТИКУМА НА ЭВМ" ДЛЯ РЕШЕНИЯ СИСТЕМ ЛИНЕЙНЫХ УРАВНЕНИЙ

1. Программа должна состоять из трех модулей - main (там находится только функция main, обеспечивающая разбор командной строки) интерфейсного (файловый ввод-вывод, выделение памяти, обработка ошибок и т.д.) и вычислительного (обеспечивающего собственно решение задачи). main-модуль должен находиться в файле SS_C_NNmain.c, интерфейсный - SS_С_NNintf.c, вычислительный - SS_С_NNtask.c; где SS - 2 цифры - номер студента в списке, С- номер раздела (в данном случае 1), NN - 2 цифры - номер задачи в разделе. Модули должны быть написаны на ANSI C, разрешается использование только библиотек stdio.h, stdlib.h, math.h. Использование других заголовков и языка C++ запрещается.

2. Программа должна получать входные данные через файл и записывать решение тоже в файл. Имена этих файлов передаются через параметры командной строки, рекомендуемый синтаксис

<program_name> input_file_name output_file_name

В случае, если параметры командной строки опущены, программа должна использовать имена файлов по умолчанию: SS_С_NNin.txt - входной файл, SS_С_NNout.txt - выходной файл. Программа должна корректно диагностировать ошибки записи и чтения из файлов, информировать о них пользователя через стандартный поток ошибок и корректно завершать работу. Результатом работы программы должен быть 0 в случае успешной работы либо код ошибки в случае ошибки.

3. Формат входного файла. Для систем с матрицей общего вида или симметрических:

   n
   a_1_1 a_1_2 ... a_1_n
   a_2_1 a_2_2 ... a_2_n
   .....................
   a_n_1 a_n_2 ... a_n_n
   b_1
   b_2
   ...
   b_n

Здесь n - число уравнений в системе (число неизвестных всегда считаем равным числу уравнений), a_i_j - элементы матрицы системы, b_i - элементы вектора правой части.

Для систем с трехдиагональной матрицей:

   n
   a_1 c_1
   d_1 a_2 c_2
   d_2 a_3 c_3
   ...........
   d_{n-2} a_{n-1} c_{n-1}
   d_{n-1} a_n
   b_1
   b_2
   ...
   b_n

Здесь n - число уравнений, a_i - элементы главной диагонали, c_i - элементы верхней диагонали, d_i - элементы нижней диагонали, b_i - элементы вектора правой части.

4. Формат выходного файла. Если решение существует (для недоопределенных систем свободные неизвестные полагать нулями)

   n
   x_1
   x_2
   ...
   x_n

Если решения не существует, выходной файл должен содержать одно число  '0'.
Если метод неприменим к данной матрице, выходной файл должен содержать одно число '-1'

5. Решение системы линейных уравнений должно быть оформлено в виде подпрограммы, находящейся в отдельном файле и имеющей следующий заголовок:

   int task_SS_С_NN_solve(int n, double* A, double* B, double* X, double* tmp)

Здесь A - массив с матрицей системы в формате a_1_1 a_1_2 ... a_1_n a_2_1 a_2_2 ... a_n_n для матрицы общего вида и a_1 a_2 ... a_n c_1 c_2 ... c_n d_1 d_2 ... d_n для трехдиагональной матрицы, B - массив правых частей, X - массив решения, tmp - массив дополнительной памяти (если она требуется методу). Получать в этой подпрограмме дополнительную информацию извне через глобальные переменные, общие блоки, включаемые файлы и т. п. запрещается. Выделение памяти в подпрограмме также запрещается; вся дополнительная память должна передаваться через массив tmp. При этом в документации на программу должна явно указываться формула, определяющая размер необходимой дополнительной памяти в зависимости от n.

Подпрограмма решения системы должна возвращать следующие значения:

   0 - работа завершена успешно, решение построено
   1 - работа завершена успешно, решения системы не существует
   -1 - метод решения не применим к данной системе

Метод решения может быть не применим к системе, например, в силу вырожденности матрицы системы для метода решения через построение LU разложения.

Метод решения должен быть реализован таким образом, чтобы он допускал решение максимально широкого класса систем. Например, для всех вариаций метода Гаусса недоопределенные системы должны решаться всегда; метод ортогонализации должен проходить для любых систем с невырожденной матрицей.
